프로젝트 개요
업비트의 이더리움 분봉 데이터를 가지고 앞으로의 가격이 어떻게 변할지 예측하는 모델을 만들어보았습니다. 처음에는 단순히 코딩만 하면 끝날 줄 알았는데 금융 데이터는 생각보다 훨씬 복잡하고 까다로웠습니다. 이론으로만 보던 시계열 데이터를 직접 다뤄보며 모델이 학습할 수 있는 형태로 데이터를 가공하는 과정이 얼마나 중요한지 뼈저리게 느꼈습니다.

데이터 전처리
처음 CSV 파일을 불러왔을 때 90만 행이 넘는 데이터를 보고 이걸 어떻게 처리해야 하나 손도 못 대고 한참을 멍하니 있었습니다. 그냥 가격 데이터만 넣었더니 예측이 전혀 안 되는 걸 보고 실제 투자자들이 보는 지표들이 필요하겠다는 생각이 들었습니다. 그래서 보조 지표들을 하나씩 직접 계산해서 모델에 추가해 주었습니다.

이동평균선(MA): 가격의 전체적인 흐름을 파악하기 위해 5분과 20분 평균 가격을 계산해서 추세를 비교해 보았습니다.

RSI(상대강도지수): 지금 시장이 너무 과열되었는지 아니면 너무 가라앉았는지를 수치로 판단할 수 있게 지표를 넣었습니다.

볼린저 밴드: 가격이 통계적인 범위 내에서 어느 위치에 와 있는지 파악하기 위해 상하단 밴드와 현재가의 비율을 계산했습니다.

수익률 및 변동성: 바로 직전 시간과 비교해서 얼마나 변했는지, 그리고 고가와 저가의 차이를 이용해 변동성 지표를 만들었습니다.

모델링 및 성능 개선 과정
가장 먼저 노드에서 본 대로 랜덤 포레스트를 돌려봤는데 점수가 너무 낮게 나와서 정말 당황스러웠습니다. 포기하고 싶기도 했지만 성능을 올리기 위해 여러 방법을 시도했습니다.
처음에는 모델만 바꾸면 점수가 오를 줄 알았는데 설계를 잘못해서 점수가 오히려 떨어지는 순간도 겪었습니다. 이를 해결하기 위해 테스트 세트 구간을 고정하고 지표가 전혀 없는 기본 모델과 지표를 쏟아부은 개선 모델을 대조하는 방식으로 다시 설계했습니다. 90만 건의 데이터를 모두 돌리기엔 시간이 너무 오래 걸려 효율적으로 샘플링하여 학습을 진행하는 법도 고민하며 진행했습니다.
처음 목표였던 5% 향상에 한참 못 미쳐서 걱정이 많았는데 지표를 추가하고 튜닝을 거치면서 특히 보조 지표가 들어갔을 때 모델의 성능이 확연히 좋아지는 것을 수치로 확인할 수 있었습니다.

결론
이번 프로젝트는 솔직히 넘기 힘든 높은 벽처럼 느껴졌습니다. 90만 개의 데이터를 처리하는 과정에서 발생하는 수많은 에러와 생소한 파이썬 라이브러리들을 다루며 스스로의 부족함을 많이 느꼈고 막막함에 포기하고 싶은 순간도 많았습니다.
뭐라도 시도해보자는 마음으로 다양한 자료와 코드들을 찾아보았습니다. 특히 이해하기 어려웠던 지표들이 실제 가격 차트에서 어떻게 움직이는지 시각화하며 하나씩 검증해 나가며 단순히 코드를 실행하는 것을 넘어 데이터의 흐름을 읽는 법을 배우는 경험이 되었습니다. 이번 프로젝트를 통해 막연했던 데이터 분석의 과정에 대해 조금이나마 감을 잡을 수 있게 되었습니다.
